# The Full Flow Challenge

A complete scenario combining all the jj techniques you've learned: commits, bookmarks, conflicts, history editing, and rebasing.

## Scenario

You're implementing a new feature for a project. You'll start from scratch, create a feature branch, make commits, encounter and resolve a conflict, edit history, and rebase onto main.

## Exercise 1: Initialize repository and make initial commits

Set up a new repository with base commits.

<TryIt command="jj git init" description="Initialize a new jj repo" />

<TryIt command="echo '# My Project' > README.md" description="Create README" />

<TryIt command="jj describe -m 'Add README'" description="Name commit" />

<TryIt command="jj new" description="Commit" />

## Exercise 3: Edit history and resolve conflicts

Modify earlier commits and handle any conflicts that arise.

<TryIt command="jj edit @-" description="Move to parent commit" />

<TryIt command={`echo 'export const HOST = "localhost"' >> config.js`} description="Add host config" />

<TryIt command="jj describe -m 'Add host config'" description="Name edited commit" />

<TryIt command="jj new" description="Create new @ commit" />

## Exercise 4: Rebase feature and complete workflow

Rebase your feature onto main and finalize.

<TryIt command="jj rebase -d main --source feature-" description="Rebase feature onto main" />

<TryIt command="jj log" description="Verify rebased history (6+ commits)" />

<TryIt command="jj status" description="Check final state" />

<TryIt command="jj bookmark list" description="Verify bookmarks are correct" />

## Key Points

- `jj git init` creates a new repository
- Bookmarks track branch points
- `jj edit @` lets you modify past commits
- `jj rebase -d` moves commits to new parents
- History is mutable and always correctable

## Completion Checklist

- [ ] Initialized repository with 3+ base commits
- [ ] Created and verified feature bookmark
- [ ] Edited commit history and resolved conflicts
- [ ] Rebased feature onto main (6+ commits total)
- [ ] Verified clean working copy state
